# L27/C05/T01 — 3-2-1 Backup Rule

## Learning Objectives

- Apply 3-2-1
- Layered backup

## 3-2-1 Rule

- **3** copies of data
- **2** different media
- **1** off-site

Original on disk. Snapshot. S3 cross-region.

## Why

- Disk failure: backup
- Backup corrupted: another
- Datacenter loss: off-site

## In Cloud

- Primary: prod DB
- Snapshot: same region
- Cross-region copy: S3

Plus:
- Glacier archive
- Cross-cloud (rare)

## Versions

3-2-1-1-0 (modern):
- 3 copies
- 2 media
- 1 off-site
- 1 immutable (ransomware)
- 0 errors (tested)

## Media Diversity

- Disk + S3
- S3 + Glacier
- Local + cloud

For: defense against media failure.

## Off-Site

Definition:
- Different physical location
- Different region
- Different cloud

For: catastrophic loss.

## Frequency

- Continuous (WAL)
- Daily (full)
- Weekly (long retention)
- Monthly (archives)

Per data class.

## Retention

- 7 days: short
- 30 days: standard
- 1 year: long-term
- 7 years: compliance

## Test

3-2-1-1-0 includes 0 errors:
- Test restores
- Monthly

(See T02.)

## Real

For DB:
- Live primary
- Snapshot
- Cross-region S3
- Glacier archive
- Tested monthly

## Best Practices

- 3-2-1 minimum
- 3-2-1-1-0 modern
- Immutable for critical
- Tested

## Common Mistakes

- One copy (not backup)
- Same region only
- Not tested
- No immutable

## Quick Refs

```
3 copies
2 media types
1 off-site
+1 immutable
+0 errors (tested)
```
