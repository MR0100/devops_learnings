# L29/C03/T01 — The 35-Minute Framework

## Learning Objectives

- Structure system design
- Pace 45-min round

## Time Box

```
0-5 min:   Clarify requirements
5-10 min:  Capacity estimation
10-15 min: High-level design (4 boxes)
15-25 min: Deep dive on components
25-35 min: Scale + reliability + trade-offs
35-40 min: Bottlenecks, future
40-45 min: Q&A
```

## 0-5: Clarify

Ask:
- Use cases (functional)
- Scale (non-functional)
- Constraints

Show:
- Mature engineer
- Don't assume

## 5-10: Estimate

Math:
- QPS
- Storage
- Bandwidth

Show numbers.

## 10-15: High-Level

Sketch:
- Client
- App
- Data
- Infra

Boxes + arrows.

## 15-25: Deep Dive

Per component:
- Tech choice
- Justification
- Trade-offs

## 25-35: Scale + Reliability

- Sharding
- Replication
- Caching
- Failure modes
- HA / DR

## 35-40: Bottlenecks

- Identify
- Mitigate

## 40-45: Q&A

- Their questions
- Your questions

## Communication

- Drive the discussion
- Don't just answer
- Show thought process

## Drive

Don't:
- Wait for prompts

Do:
- Lead through framework
- Ask "should I cover X next?"
- Manage time

## Whiteboard / Tool

- Excalidraw common
- Practice drawing

## Common Mistakes

- Jump to design (no clarify)
- One area too long
- Skip scale
- No trade-offs

## Best Practices

- Practice with timer
- Mock interviews
- Multiple problems
- Patterns down

## Quick Refs

```
Clarify → Estimate → High-level → Deep dive → Scale → Bottlenecks → Q&A

Time: 5 / 5 / 5 / 10 / 10 / 5 / 5

Drive: don't wait
```
