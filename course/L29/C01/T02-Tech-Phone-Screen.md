# L29/C01/T02 — Tech Phone Screen

## Learning Objectives

- Pass tech phone screen
- Gate to onsite

## Format

45-60 minutes:
- Brief intro (5 min)
- Tech question (35-45 min)
- Your questions (5-10 min)

## Question Types

For DevOps / SRE roles:

### Coding
LeetCode-style:
- Easy to medium
- 1-2 problems

### Linux / Systems
- Debug a script
- Performance issue
- Bash / scripting

### System Design (mini)
- Smaller system
- 20-30 min

### Conceptual
- "Walk through what happens when you `kubectl apply`"
- Deep dive

## Prep

- Practice LeetCode (med)
- Review CS fundamentals
- Be ready to think aloud

## Communication

- Talk through approach
- Ask clarifying questions
- Edge cases
- Test with example

## Common Mistakes

- Silent thinking
- Jump to code
- No examples
- Miss edge cases

## Stress

For phone screen:
- Stakes medium
- Don't choke
- Practice mock

## After

- Recruiter follows up 1-7 days
- Pass: onsite
- Fail: feedback (sometimes)

## Best Practices

- Mock interviews
- Out loud
- Edge cases
- Ask questions
