# L29/C01 — The FAANGM Interview Process

## Topics

- **T01 Recruiter Screen** — 30 min. Confirm background, interest, level fit, compensation expectations. Be vague on comp expectations until offer.
- **T02 Tech Phone Screen** — 60 min coding or system design lite. Need to nail this to get to onsite.
- **T03 Onsite Loop (Coding, System Design, Behavioral)** — 4-6 hrs. 5-7 rounds. Each round is a vote.
- **T04 Bar Raiser / Hiring Committee** — Amazon's bar raiser is *outside* the team and trained. Google has a hiring committee that reviews packets.
- **T05 Levels at Each Company** — Match yourself to the right level before applying.

## Loop Composition (Typical Senior+ DevOps)

| Round | Focus | Duration |
|---|---|---|
| Coding 1 | Medium DSA | 45 min |
| Coding 2 | Medium/hard DSA OR systems coding | 45 min |
| System Design 1 | Standard distributed system | 60 min |
| System Design 2 | Domain-specific (often DevOps/platform) | 60 min |
| Behavioral / Leadership | LP/Googleyness/Pillars | 45-60 min |
| Deep technical | Domain (K8s, AWS, networking) | 60 min |
| Bar Raiser / Hiring Mgr | Mixed | 45-60 min |

## Outcomes

Each interviewer votes:
- Strong Hire / Hire / Lean Hire
- No Hire / Strong No Hire

One Strong No can sink the loop. Two No Hires usually do.

## Calibration: Strong Hire Signals

- Solves the problem completely with clean code
- Drives system design with confident tradeoff discussion
- Behavioral stories show level-appropriate scope and impact
- Asks clarifying questions; clearly explains thinking
- Owns mistakes, recovers gracefully

## Failure Modes to Avoid

- Hesitation before starting
- Talking instead of coding
- Failing to test code (run through it)
- Forgetting edge cases
- Defending bad approaches when challenged
- Not listening to hints
- Asking too few clarifying questions
- Generic stories ("we improved efficiency")
- Trash-talking previous employers

## Final Stage: Hiring Committee

- Amazon Bar Raiser is independent of hiring team
- Google packet reviewed by committee (not hiring manager)
- Meta has calibration discussions
- Microsoft is more team-decision
- These are checks against pressure-based bad hires

## Interview Themes

- Know the company's specific process (recruiter will share)
- Practice each round type independently
- Mock interviews are non-optional at this level
