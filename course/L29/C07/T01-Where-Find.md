# L29/C07/T01 — Where to Find Mock Interviews

## Learning Objectives

- Get practice interviews
- Use feedback

## Sources

### Pramp
Free; peer-to-peer.

### interviewing.io
Anonymous; senior engineers.
Free + paid tiers.

### Codility
Self-paced + coaching.

### Friends
Engineering friends.
Cheap; biased sometimes.

### Coaches
Paid; high quality.
- Career Hub
- Pathrise
- Independent

### Bootcamps
Some include mocks.

### Yourself
Whiteboard alone; record + review.

## Frequency

For onsite prep:
- 10-20 mocks total
- Weekly initially
- Daily near interview

## Per Round Type

- Coding: most
- System design: 5-10
- Behavioral: 3-5

## Get Feedback

- What went well
- What to improve
- Specific actions

## Iterate

- Mock
- Feedback
- Practice
- Mock again

## Time

Pre-interview prep:
- 1-2 months
- Mock weekly
- Real interviews start: rotate

## Tools

- Excalidraw (whiteboarding)
- CoderPad (coding)
- Zoom / Meet (video)

## Best Practices

- Treat like real
- Time-bound
- Talk through
- Take notes after

## Common Mistakes

- Skip mocks
- Friends only (soft)
- No feedback
- Don't iterate

## Quick Refs

```
Sources:
- Pramp (free peer)
- interviewing.io (anonymous)
- Coaches (paid quality)
- Friends (limited)

Frequency: weekly → daily
```
