# L29/C07/T02 — How to Self-Score

## Learning Objectives

- Self-evaluate
- Find weaknesses

## Rubric

After each mock:

### Coding
- Approach clear?
- Time efficient?
- Code clean?
- Edge cases?
- Tested?
- Complexity discussed?

### System Design
- Clarified requirements?
- Estimated?
- High-level clear?
- Deep dive solid?
- Scale addressed?
- Drove discussion?

### Behavioral
- STAR structure?
- Specific?
- Quantified?
- Self-aware?
- I-focused (not we)?
- Growth mindset?

Score 1-5 each.

## Bar by Level

| | Junior | Senior | Staff |
|---|---|---|---|
| Code clean | 4+ | 5 | 5 |
| Time | 4+ | 5 | 5 |
| Edge cases | 3+ | 4+ | 5 |
| System scale | 2+ | 4+ | 5 |
| Trade-offs | 1+ | 4+ | 5 |
| Drive | 1+ | 4+ | 5 |

For: stretch to next level.

## Patterns

Across mocks:
- Same weaknesses?
- Focus there

For example:
- "Always skip edge cases"
- "Never quantify behavioral"

For: target.

## Record

```
Mock 1 (2026-01-10): 
  Coding: 3/5 (skipped edge case)
  Time: 4/5 (rushed last 5 min)
  
Action: Practice 5 LeetCode med with strict timing.
```

## Improve

Per weakness:
- Targeted practice
- Re-mock
- Compare

For: progress.

## Confidence

Watch:
- Improving over time
- Bar by level
- Diversity of problems

## Best Practices

- Honest scoring
- Specific actions
- Track over time
- Iterate

## Common Mistakes

- Generous self-scoring
- Vague actions
- Don't track
- No iteration

## Quick Refs

```
Score 1-5 per dimension
Patterns across mocks
Targeted improvement
Track progress
```
