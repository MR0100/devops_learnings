# L29/C08/T01 — Impact-Driven Bullets

## Learning Objectives

- Write resume bullets
- Show impact

## Bad Bullet

"Maintained Kubernetes clusters."

Why bad:
- No impact
- No scale
- No metrics

## Good Bullet

"Led migration of 50+ microservices to Kubernetes, reducing deployment time 80% (45min → 9min) and infrastructure cost $200k/year."

Why good:
- Specific (50+ services)
- Quantified (80% reduction)
- Cost impact ($200k)
- Action verb (Led)

## Pattern

```
Action verb + What + Scale + Quantified outcome + (Why mattered)
```

## Action Verbs

Strong:
- Led, Drove, Architected, Designed
- Reduced, Improved, Increased, Optimized
- Built, Migrated, Automated
- Mentored, Hired, Promoted

Avoid:
- Worked on, Helped, Was responsible for

## Quantify

- Cost: $X saved
- Time: Y% faster
- Scale: N users / services / requests
- Quality: 99.99% from 99.5%

For: every bullet.

## Examples

### Reliability
"Improved payments service availability from 99.5% to 99.97% (P99 latency: 800ms → 200ms), enabling 4× transaction growth."

### Cost
"Reduced AWS spend $1.2M/year by right-sizing 200 EC2 instances and migrating to Spot."

### Speed
"Architected GitOps deployment platform reducing prod deploys from days to 15 min for 30 engineering teams."

### Leadership
"Mentored 4 engineers; 2 promoted to Senior within 18 months."

## Show Senior

For senior+:
- Lead / architect / drive
- Cross-team
- Strategic

Not:
- Just doing tasks
- Sub-scope

## ATS-Friendly

- Keywords (K8s, Terraform, AWS, SRE, etc.)
- Standard formatting
- No fancy fonts / columns

(See T02.)

## Length

- One page (< 10 yr exp)
- Two page max
- Tight

## Best Practices

- Impact first
- Quantify
- Action verbs
- ATS-friendly

## Common Mistakes

- Tasks not impact
- No metrics
- Weak verbs
- Too long

## Quick Refs

```
Action + What + Scale + Outcome
Quantify everything
Strong verbs
One page (usually)
```
